Features

Everything Claude needs.
Nothing it doesn't.

🌳
AST-Aware Chunking

Tree-sitter parses Python, JavaScript, TypeScript, PHP, Go, Rust, and Java into semantic chunks. Functions, classes, imports. No raw file dumps, no context waste.

⚡
Hybrid Search + Graph Expansion

Vector similarity + BM25 keyword search merged via Reciprocal Rank Fusion. Then CCE walks one hop on the code graph — if auth.py is a hit, utils.py it imports comes too.

🧠
Smart Compression

With Ollama running locally, chunks are summarized by phi3:mini. Without it, smart truncation extracts signatures and docstrings. Four output levels: off, lite, standard, max.

🔗
Session Memory

Recall decisions and code areas across sessions via session_recall, record_decision, and record_code_area. No re-explaining your architecture every session.

🔄
Live File Watcher

Files are re-indexed automatically on save during cce serve. Git hooks handle commits. Debounced, deduped, zero manual effort.

📋
Project Commands

Persist rules, preferences, and shell hooks per project with cce commands. "Never use down() in migrations" — stored once, surfaced to Claude every session.

🎨
Colorful CLI

Every command outputs styled, animated results. Section headers, colored bullets, line-by-line reveal. Run cce for a 2-column status dashboard in your terminal.

📊
Web Dashboard

Run cce dashboard for a Grafana-inspired dark theme with donut charts, bar graphs, file health, and live 5-second polling. All in a single embedded HTML file.

Methodology

For each query, "without CCE" is the full content of every file the query touches (what an AI agent would read). "With CCE" is only the relevant code chunks returned by context_search.

This is conservative. Without CCE, AI agents typically read more files than needed because they don't know which are relevant upfront.

Under the hood

Clever engineering.
Zero magic.

Every decision optimizes for speed, cost, or reliability. Here is what makes CCE tick.

⚡
Content-Hash Embedding Cache

SHA-256 fingerprint per chunk, salted with model name. Re-index skips unchanged code. Binary-packed embeddings (struct.pack) are 10x smaller than JSON. Typical re-index: 96% cache hit, under 1 second.

🔀
Hybrid Retrieval + RRF

Vector search (semantic) and BM25 (exact keyword) run in parallel. Merged via Reciprocal Rank Fusion (K=60). Confidence blends similarity (50%), keyword match (30%), and recency (20%). Code-lookup queries boost FTS 1.5x.

🪶
sqlite-vec: 2 MB, not 217 MB

Replaced LanceDB with sqlite-vec. Same search quality, 99% smaller install. WAL mode + PRAGMA NORMAL for 80% write speedup. Vectors, FTS5, code graph, compression cache all in three SQLite files.

📐
Deterministic Grammar Compression

No LLM needed. Drops articles, fillers, pronouns from memory entries. Three levels (lite/full/ultra, 20-60% savings). Code, paths, URLs preserved byte-for-byte. Same input always yields same output.

🔒
Fail-Closed Hooks

5 Claude Code lifecycle hooks capture session context. Every hook runs curl ... || true so a crashed server never blocks the user. SessionStart injects bootstrap context; others capture silently.

💰
Dynamic Pricing

cce savings shows dollar estimates from live Anthropic pricing (fetched, cached 7 days, offline fallback). No manual updates when rates change. Per-bucket breakdown: retrieval, compression, grammar, memory.

🛡
Security by Default

Path traversal protection on all MCP inputs. PII scrubbing (emails, IPs, SSNs, credit cards) on memory writes. Secret file detection (.env, *.pem) during indexing. Audit log with hashed queries.

📊
Append-Only Savings Ledger

7 buckets track every token saved: retrieval, chunk compression, output compression, memory recall, grammar, turn summarization, progressive disclosure. Survives restarts. Powers CLI and dashboard analytics.
